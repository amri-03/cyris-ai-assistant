# Axis Memory Structure

## Overview

The memory system is one of the core foundations of Axis AI Assistant.

The purpose of memory is not simply storing information, but gradually building a persistent understanding of the user's:

* priorities
* behavioral patterns
* routines
* interests
* struggles
* consistency cycles
* long-term goals

Axis should behave as a system that remembers and adapts over time.

---

# Memory Categories

## 1. Long-Term Memory

Stores:

* important goals
* recurring ambitions
* stable interests
* long-term priorities
* career direction
* recurring struggles
* preferred work styles

This memory changes slowly over time.

Examples:

* interest in cybersecurity
* desire for financial independence
* recurring inconsistency problems

---

## 2. Short-Term Context Memory

Stores:

* current focus areas
* temporary priorities
* active projects
* upcoming deadlines
* recent progress
* temporary schedule changes

This memory changes frequently.

Examples:

* current semester workload
* internship preparation
* active learning focus

---

## 3. Behavioral Memory

Stores:

* productivity patterns
* inactivity periods
* recurring avoidance
* consistency history
* momentum cycles
* focus windows

This memory helps Axis understand behavioral trends.

Examples:

* abandoning projects after excitement spikes
* working better during late evenings
* avoiding unclear tasks

---

# Memory Philosophy

The memory system should:

* reduce repeated explanations from the user
* preserve continuity
* improve contextual understanding
* support adaptive prioritization
* improve behavioral guidance

Memory should help the assistant become more useful over time.

---

# Important Constraints

The memory system should avoid:

* invasive behavior
* excessive tracking
* unnecessary complexity
* storing meaningless information
* overwhelming the user

Axis should prioritize useful understanding rather than maximum data collection.
