# Important Architecture Decisions

## Adaptive Intelligence Coordination

Cyris now coordinates:

* memory
* context
* behavior
* priorities
* guidance
* evolving state

through unified orchestration flows instead of isolated systems.

---

## Lightweight Persistence Strategy

JSON-based persistence was selected temporarily because it:

* reduces early complexity
* supports persistence experimentation
* avoids premature database complexity
* supports continuity architecture development

This persistence layer is intended to prepare future PostgreSQL integration later.

---

## Separation Of Concerns

Persistence logic was separated from memory orchestration through:

* StorageService
* MemoryManager

This improves:

* maintainability
* scalability
* future storage flexibility
