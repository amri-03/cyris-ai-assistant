# Cyris Project Rules

## Development Philosophy

Cyris should be developed:

* sequentially
* in small achievable phases
* with strong documentation continuity
* with architecture clarity before complexity
* with momentum preservation as a priority

## Core Principles

* Intelligence-first architecture over UI-first development
* Behavioral understanding before advanced automation
* Minimal manual input whenever possible
* Context-aware assistance instead of productivity pressure
* Adaptive prioritization instead of static task management
* Calm and practical guidance instead of motivational coaching

## MVP Constraints

The MVP should NOT include:

* voice assistant systems
* autonomous desktop control
* complex agents
* excessive dashboards
* advanced analytics systems
* unnecessary AI hype features
* large-scale automation

## Documentation Rules

* Every meaningful phase must be documented
* Important architectural decisions must be recorded
* Development continuity must be preserved
* Daily logs should track progress and blockers
* Fake or imagined implementations should never be documented as completed

## Development Workflow

The project should prioritize:

* controlled scope
* visible progress
* low-friction workflows
* maintainability
* realistic implementation speed

## Primary Focus Areas

Early development should focus on:

* persistent memory
* behavioral logic
* contextual understanding
* prioritization systems
* adaptive guidance
* conversational interaction
